# 🎬 日本語字幕提取 — 技術方案與優化建議

## 一、整體流程總覽

```
視頻文件
  │
  ▼
┌─────────────────────────────┐
│ Step 1: ffmpeg 音頻提取      │  視頻 → 16kHz 單聲道 WAV
└──────────────┬──────────────┘
               ▼
┌─────────────────────────────┐
│ Step 2: Demucs 人聲分離      │  混合音頻 → 純人聲軌道
└──────────────┬──────────────┘
               ▼
┌─────────────────────────────┐
│ Step 3: ffmpeg 語音增強      │  降噪 + 帶通濾波 + 響度標準化
└──────────────┬──────────────┘
               ▼
┌─────────────────────────────┐
│ Step 4: stable-ts 轉錄      │  音頻 → 帶時間戳的日文文本
└──────────────┬──────────────┘
               ▼
┌─────────────────────────────┐
│ Step 5: 字幕質量檢查         │  去幻覺/BGM歌詞/異常段
└──────────────┬──────────────┘
               ▼
┌─────────────────────────────┐
│ Step 6: 後處理 + VTT 輸出    │  拆分/合併/格式化 → .vtt
└─────────────────────────────┘
```

## 二、原始方案評估

### ✅ 做得好的地方

| 項目 | 評價 |
|------|------|
| Demucs 人聲分離 | 非常正確的選擇，動漫/影視中 BGM 和音效干擾嚴重，Demucs 是目前最佳開源方案 |
| suppress_silence | 避免靜音段拖長字幕時間，必備 |
| word_timestamps | 詞級時間戳是精確對齊的基礎 |
| condition_on_previous_text | 利用上下文提升連貫性，對日文很有幫助 |
| EBU R128 響度標準化 | 專業標準，確保 Whisper 輸入音量一致 |
| VAD 配置 | onset=0.5 / offset=0.363 是合理的默認值 |
| beam_size=10 | 比默認值 5 更精確 |

### ⚠️ 需要改進的地方（6 項關鍵問題）

---

### 問題 1：帶通濾波範圍過於激進 🔴 嚴重

**原方案**: 高通 200Hz + 低通 4000Hz

**問題分析**:
- 音頻採樣率 16kHz → Nyquist 頻率 = 8kHz
- 低通 4kHz = **砍掉了一半可用頻譜**
- 日文的清音（さしすせそ、たちつてと）的齒擦音能量集中在 **4kHz~8kHz**
- 高通 200Hz 會切掉男性基頻 (85~180Hz)

**建議修改**:
```
高通: 80 Hz   （去除低頻隆隆聲，保留男聲基頻）
低通: 7500 Hz （保留幾乎全部語音頻譜，僅去除極高頻噪音）
```

---

### 問題 2：未強制指定語言 🔴 嚴重

**問題**: 未設置 `language="ja"`，Whisper 會自動檢測語言
- 短句可能被誤判為中文/韓文
- 語言切換會導致輸出混亂

**建議**: 強制 `language="ja"`

---

### 問題 3：Whisper 模型版本 🟡 中等

**原方案**: `large`

**建議**: 使用 `large-v3`（或 `turbo`）
- `large-v3` 相比 `large-v1` 日文 CER 降低約 **15-20%**
- `turbo` 速度快 8 倍，精度接近 large-v3，推薦 CPU 用戶使用

---

### 問題 4：缺少日文專用 Initial Prompt 🟡 中等

**問題**: 沒有設置 `initial_prompt`，Whisper 可能：
- 輸出半角標點而非全角
- 漢字/假名選擇不一致

**建議**:
```python
initial_prompt = "こんにちは。この番組では、日本語の字幕を正確に書き起こします。句読点は「、」「。」を使います。"
```
這段 prompt 引導 Whisper：
1. 使用全角日文標點
2. 保持日語語境
3. 規範句讀格式

---

### 問題 5：Demucs 模型可升級 🟢 輕微

**原方案**: `htdemucs`

**建議**: 使用 `htdemucs_ft`（fine-tuned 版本）
- 人聲分離 SDR 提升約 **0.5~1.0 dB**
- 代價是推理速度略慢

---

### 問題 6：幻覺檢測不夠全面 🟡 中等

**原方案**: 過濾 BGM 歌詞/幻覺文本/零時長段

**建議加強**:
| 檢測維度 | 方法 | 閾值 |
|----------|------|------|
| 重複文本 | 連續 N 段文本相同則標記 | N ≥ 2 |
| 壓縮比 | `compression_ratio > threshold` | > 2.4 |
| 無語音概率 | `no_speech_prob > threshold` | > 0.6 |
| 對數概率 | `avg_logprob < threshold` | < -1.0 |
| 字元速率 | 日文每秒不超過 N 字 | > 15 字/秒 |
| 典型幻覺模式 | 正則匹配重複字符串 | `(.{2,})\1{3,}` |
| BGM 歌詞特徵 | 含有 ♪♫🎵 或全片假名長句 | 模式匹配 |

| 參數 | 原始值 | 優化值 | 說明 |

## 四、優化後的處理流程

```
視頻文件 (.mp4/.mkv/.avi/...)
  │
  ▼
[Step 1] ffmpeg 音頻提取
  │  → 16kHz, mono, WAV
  │  → 自動檢測音軌（選擇日語音軌）
  │
  ▼
[Step 2] Demucs htdemucs_ft 人聲分離
  │  → vocals.wav（純人聲）
  │  → 自動檢測 GPU/MPS/CPU
  │
  ▼
[Step 3] ffmpeg 語音增強
  │  → 高通 80Hz + 低通 7500Hz
  │  → 動態壓縮 (compand)
  │  → EBU R128 響度標準化 (-16 LUFS)
  │
  ▼
[Step 4] stable-ts 轉錄 (Whisper large-v3)
  │  → language="ja"
  │  → initial_prompt 引導
  │  → beam_size=10
  │  → suppress_silence=True
  │  → word_timestamps=True
  │  → temperature fallback
  │
  ▼
[Step 5] 多維度質量檢查
  │  → 壓縮比過濾 (> 2.4)
  │  → 無語音概率過濾 (> 0.6)
  │  → 對數概率過濾 (< -1.0)
  │  → 重複文本檢測
  │  → 字元速率異常檢測 (> 15 字/秒)
  │  → BGM 歌詞/幻覺模式匹配
  │  → 漏字幕時間段檢測
  │
  ▼
[Step 6] 後處理
  │  → 過短段合併 (< 0.3s)
  │  → 過長段按標點拆分 (> 8s)
  │  → 最短顯示保底 (0.5s)
  │  → 日文標點規範化
  │
  ▼
輸出: .vtt / .srt / .json
```

## 五、環境需求

### 系統依賴

```bash
# macOS
brew install ffmpeg

# Ubuntu/Debian
sudo apt install ffmpeg
```

### Python 依賴

```bash
pip install stable-ts       # Whisper + 精確時間戳
pip install demucs           # 人聲分離
pip install torch            # PyTorch (Demucs/Whisper 後端)
pip install openai-whisper   # Whisper 模型
```

### 硬件建議

| 配置 | 推薦模型 | 預計速度 |
|------|----------|----------|
| NVIDIA GPU (≥8GB VRAM) | `large-v3` | ~1x 實時 |
| Apple Silicon (M1/M2/M3) | `large-v3` (MPS) | ~2-3x 實時 |
| CPU only | `turbo` 或 `medium` | ~5-15x 實時 |

## 六、使用方式

### 基本用法

```bash
# 單個視頻
python tiqu.py video.mp4

# 指定輸出目錄
python tiqu.py video.mp4 --output-dir ./subtitles

# 使用較快的模型（CPU 推薦）
python tiqu.py video.mp4 --model turbo

# 批量處理
python tiqu.py video1.mp4 video2.mkv video3.avi

# 同時輸出 VTT 和 SRT
python tiqu.py video.mp4 --format vtt srt
```

### 進階選項

```bash
# 跳過人聲分離（音源已經很乾淨）
python tiqu.py video.mp4 --skip-demucs

# 使用 Silero VAD 預過濾
python tiqu.py video.mp4 --vad silero

# 調整幻覺過濾嚴格度
python tiqu.py video.mp4 --hallucination-threshold strict

# 保留中間文件（用於調試）
python tiqu.py video.mp4 --keep-temp
```

## 七、已知限制與注意事項

1. **多人對話**: Whisper 不做說話人識別（diarization），如需區分說話人需額外整合 pyannote-audio
2. **唱歌片段**: Demucs 分離後的人聲若包含唱歌，Whisper 可能產生幻覺，建議配合 BGM 歌詞過濾
3. **方言/口音**: Whisper large-v3 對標準日語最優，方言（関西弁等）精度會下降
4. **超長視頻**: 建議按場景分段處理，避免記憶體溢出
5. **首次運行**: 需下載 Whisper 模型（large-v3 約 3GB）和 Demucs 模型（約 80MB）
